# Contributing

## Quy tắc nhánh
- `main`: stable, chỉ nhận PR đã pass CI.
- `develop`: tích hợp.
- Feature: `feat/<short-name>`, Hotfix: `fix/<short-name>`.

## Commit message (Conventional Commits)
- `feat: add facility_count endpoint`
- `fix: handle utf8 city param`
- `chore/docs/refactor/test`

## Quy trình PR
1. Fork → tạo nhánh
2. PR kèm mô tả, kết quả.


## Coding style
- ESLint + Prettier / Black + isort (tùy stack), CI sẽ check.
